## 5.1 배열
자바에선 기본 자료형 외는 전부 참조 자료형이다.
예) 배열, 클래스, 인터페이스, 개발자가 정의한 모든 자료형
### 5.1.1 배열이란?
특징
- `동일한` 자료형을 묶어 저장하는 참조 자료형
- 생성할 때 크기를 지정해 주어야 하며, 변경할 수 없다.
### 5.1.2 배열 생성하기

#### (1) 배열 선언하기
- 방법 1. `자료형[] 변수명` (권장, 아래는 c 스타일)
- 방법 2. `자료형 변수명[]`
- 선언된 변수 공간은 생성되었지만, 객체는 생성되지 않아 비워진 상태
#### (2) 힙 메모리에 배열의 객체 생성하기
- 방법 `new 자료형[배열의길이]`

#### (3) 배열 자료형 변수에 객체 대입하기
- 방법 1. `자료형[] 변수명 = new 자료형[베열의 길이];`
- 방법 2. 
  ```
  자료형[] 변수명;
  변수명 = new 자료형[배열의 길이];
  ```
- 예시) `int a = new int[3];`
  int 자료형 3개를 저장할 수 있는 공간을 힙 메모리에 넣어두고, 그 위치를 참조 변수 a에 저장해죠!

＊ 스택 메모리 공간 vs 힙 메모리 배열 공간
-  스택 메모리 공간 : 값을 초기화 하지 않으면 빈 공간으로 존재
- 힙 메모리 공간 : 값을 안주면 빈 공간이 없다! (값을 주지 않으면 컴파일러가 다 강제 초기화 시킨다. 숫자는 0, 불리언은 false, 그 외는 null)
#### (4) 객체에 값 입력하기
- 방법 `참조할 변수명[인덱스] = 값;`
### 5.1.3 1차원 배열을 생성하는 다양한 방법
- 방법 1. 배열 객체를 생성하고 값 대입하기
  ```
  자료형[] 참조변수명 = new 자료형[배열 길이];
  참조 변수명[0] = 값;
  참조 변수명[1] = 값;
  ...
  참조 변수명[배열길이-1] = 값;
  ```
- 방법 2. 배열 객체 생성과 함께 값 대입하기 (선언과 객체 대입 분리 O)
  `자료형[] 참조변수명 = new 자료형[]{값1, 값2, 값3, ...., 값};`
- 방법 3. 대입할 값만 입력하기 (선언과 객체 대입 분리 X)
  `자료형[] 참조변수명 = {값1, 값2, 값3, ...., 값};

### 5.1.4 참조 변수와 배열 객체의 값 초기화하기
모든 변수는 스택 메모리에 저장된다.
스택 메모리는 변수를 초기화하지 않으면 출력할 때 오류가 발생한다.
힙의 기본 자료형 변수 : 초깃값으로 0, false 등을 저장한다.
힙의 참조 자료형 변수 : 주소를 가리켜야 하는데 위치가 없으므로, null을 출력한다.
### 5.1.5 참조 자료형으로서 배열의 특징
- 기본 자료형 변수 복사 = 값 복사
- 참조 자료형 변수 복사 = 값이 저장된 위치 복사 
### 5.1.6 반복문을 이용해 배열 데이터 읽기
- 방법 1. `배열 참조 변수.length` 사용해서 반복문 횟수 구하고 반복하여 읽기
- 방법 2. for-each문 사용하기 (집합 객체에서 원소를 꺼내는 방식)
  `for(자료형 변수명: 집합 객체) {}`
- 방법 3. Arrays 클래스의 toString() 정적 메서드 사용하기
### 5.1.7 2차원 정방 행렬 배열

- 선언 방법 1. `자료형[][] 변수명 = new 자료형[행][열];`
- 선언 방법 2. `자료형 변수명[][] = new 자료형[행][열];`
- 선언 방법 3. `자료형[] 변수명[] = new 자료형[행][열];`

메모리는 2차원 데이터를 바로 저장할 수 없으므로, 아래와 같은 방법을 사용한다.

- 방법 1. 배열 객체를 생성하고 값 대입하기
  ```
  자료형[][] 참조변수명 = new 자료형[행][열];
  참조변수명[0][0] = 값;
  참조변수명[0][1] = 값;
  ...
  참조변수명[행-1][열-1] = 값;
  ```

- 방법 2. 배열 객체의 행 성분 부터 생성하고 열 성분 생성하기
  ```
    자료형[][] 참조변수명 = new 자료형[행][]; //열 수를 작성하지 X//
    참조변수명[0] = 1차원 배열1;
    참조변수명[1] = 1차원 배열2;
    ...
    참조변수명[행의 길이 -1] = 1차원 배열n;
    
    /* 예상 실행 결과
    [1차원 배열 1]
    [1차원 배열 2]
    ...
    [1차원 배열 n] */*
    
  ``` 

- 방법 3. 배열의 자료형과 함꼐 대입할 값 입력하기
  ```
  자료형[][] 참조변수명 = 자료형[][] {{값, 값, 값, ..., 값}, {값, 값, 값, ... 값}};
  ```

- 방법 4. 대입할 값만 입력하기

### 5.1.8 2차원 비정방 행렬 배열
사실 데이터 테이블 같이 정방 개념이 아니라, 하나의 배열의 값이, 다른 배열의 주소를 저장할 수 있는 개념이다. 

- 방법 1. 배열 객체의 행 성분부터 생성하고 열 성분 생성하기 (위 정방 행렬의 생성 방식 2번과 같다.)
- 방법 2. 자료형과 대입할 값만 입력하기 (위 정방 행렬의 생성 방식 3번과 같다.)
- 방법 3. 대입할 값만 입력하기 (위 정방 행렬의 생성 방식 4번과 같다.)

### 5.1.9 2차원 배열의 출력
기본적으로는 1차원 배열의 출력과 크게 다르지 않으나, 2중 for문을 사용해야한다.

### 5.1.10 main() 메서드의 입력 매개 변수
`public static void main(String[] arg) {}`
- 위 파일을 실행할 때 arg에 인자값을 넣어주면 그 값을 받아서 실행이 된다. 이 값들은 공백 문자로 구분이 된다.
- 출력 방식 1. `for 문을 돌면서 args[i]로 하나씩 불러오기`
- 츨력 방식 2. `for (String 참조변수명: args) {참조변수} 형태로 하나씩 불러오기`


## 5.2 문자열을 저장하는 String
### 5.2.1 문자열의 표현과 객체 생성
- 문자열은 반드시 큰따옴표 안에 표기 
- 생성 방법 1.  `Sting 참조변수명 = new String "문자열";`
- 생성 방법 2. `Sting 참조변수명 = "문자열";`

### 5.2.2 String 클래스의 2가지 특징
- 특징 1. 한번 정의된 문자열은 변경될 수 없다.
- 특징 2. 문자열 리터럴을 바로 입력해 객체를 생성할 때(= new 객체로 생성되지 않은 경우), 이미 힙 메모리에 똑같이 리터럴로 생성된 같은 문자열끼리 객체를 공유한다.

### 5.2.3 String 객체의 '+' 연산
- 유형 1. 문자열 + 문자열 연산
- 유형 2. 문자열 + 기본자료형 연산

### 5.2.4 String 클래스의 주요 메서드
- 문자열 길이 : length()
- 문자열 검색 : indexAt(), indexOf(), lastIndexOf()
- 문자열 반환 : String.valueOf()
- 문자열 연결 : concat()
- 문자열 배열 변환 : getBytes(), toCharArray()
- 문자열 대소문자 변환 : toLowerCase(), toUpperCase()
- 문자열 수정 : replace(이전, 이후)
- 문자열 분할 : substring(), split()
- 공백 제거 : trim()
- 문자열 내용 비교 : equals(), equalsIgnoreCase()
