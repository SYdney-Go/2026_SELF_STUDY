## 6.1 클래스와 객체의 개념
### 6.1.1 클래스의 개념 알아보기
- 객체 : 사용할 수 있는 실체
- 클래스 : 객체를 만들기 위한 설계도

프로그래밍 문법 요소의 발전
```
변수 → 배열 → 구조체 → 클래스
```
- 변수 : 다양한 데이터를 저장하기 위한 요소
- 배열 : 많은 데이터 수를 관리하기 위한 요소
- 구조체 : 다양한 자료형의 배열을 관리하기 위한 요소
- 클래스 : 데이터를 넘어, 기능을 묶어 함께 관리하기 위한 요소
	- 데이터는 속성 또는 필드라고 한다
	- 기능은 메서드라고 한다.

### 6.1.2 절차지향과 객체지향 이해하기
- 절차지향형 프로그램 : 순서에 맞춰 실행되는 방식
- 객체지향형 프로그램 : 객체 단위에 맞춰 실행되는 방식

자바가 제공하는 객체지향 문법 요소
```
클래스
├ 일반 클래스
└ 추상 클래스
	└ 인터페이스 (추상 클래스 중 종특)
```

### 6.1.3 클래스 구조 살펴보기
```
package
import
class {}

public class 클래스명{
	필드
	메서드
	생성자
	이너 클래스
}
```
- 클래스 외부
	- package : 
	- import : 다른 패키지의 클래스를 사용할 수 있게 불러오는 기능
	- class : 클래스 외부의 클래스
- 클래스 내부
	- 필드 :  클래스의 속성을 나타내는 변수
	- 메서드 : 클레스의 기능을 담은 함수
	- 생성자 : 클래스의 객체를 생성하는 역할
	- 이너 클래스 : 클래스 내부의 클래스

### 6.1.4 클래스와 객체 구분하기
- 클래스를 직접 사용할 수 없다.
- 클래스 내 생성자로 객체를 만드는 인스턴스화 과정을 통해, 객체(인스턴스)를 만든다.
- 이렇게 만든 객체의 필드, 메서드, 이너 클래스를 사용할 수 있다.

## 6.2 객체의 생성과 활용
### 6.2.1 객체 생성하기
- 생성 방법 : `클래스명 참조변수명 = new 생성자();`
- 생성된 객체는 힙 메모리로 가서 저장되고, 그 위치 값은 참조변수가 저장하고 있다.

객체 생성에 따른 메모리 구조
```
// (예시) //
A a = new A();
```
- 클래스 A → 메모리의 클래스 영역으로 이동
- 참조변수 a → 메모리의 스택 영역으로 이동
- new → 메모리의 힙에 새로운 객체 생성
- A() → 생성자가 A라는 객체를 생성한다.
	- 필드 : 객체 내부에 저장공간을 가지고 있다.
	- 메서드 : 메서드는 객체 외부에 저장되어있으며, 객체들은 같은 기능 주소를 가진다.
	- 이너 클래스

### 6.2.2 객체 활용하기

포인트 연산자 사용하기
자바에서는 힙 메모리에 직접 접근할 수 없다. → 오직 참조 변수를 이용해서만 객체에 접근할 수 있으며, 이를 위해 포인트 연산자를 사용한다.
```
참조변수명.필드명
참조변수명.메서드명()
```
