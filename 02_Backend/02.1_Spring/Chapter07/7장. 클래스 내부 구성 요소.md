## 7.1 필드
`클래스에 포함된 변수. 객체의 속성값을 지정한다.`
### 7.1.1 필드와 지역 변수의 구분
- 지역변수 vs 필드 : 선언된 중괄호 위치
	- 지역변수 : 메스드 중괄호 안에 존재,  스택 메모리 안에 저장
	- 필드 : 클래스 중괄호 안에 존재. 힙 메모리 객체 내부에 생성

### 7.1.2 필드와 지역 변수의 초깃값
- 필드 : 힙메모리 안에 있으므로 강제 초기화 됨
- 지역변수 : 스택 메모리 안에 있으므로 강제 초기화되지 않음
## 7.2 메서드
`클래스의 기능에 해당하는 요소`

### 7.2.1 메서드 정의하기

```
자바 제어자 리턴타입 메서드명(입력매개변수){
	메서드 내용
};
```
- 리턴 타입 : 메서드 종료 이후 변환되는 값의 자료형
- 입력매개변수 : 메서드를 호출할 때 전달되는 자료형과 지역변수명 지정
- 리턴 타입이 void가 아니면 메스드 안에 반드시 return 값이 필요하다.

### 7.2.2 여러 리턴 타입의 메서드 살펴보기

타입 1. 리턴 타입이 void + 입력 매개변수 X
```
void print() {
	System.out.println("안녕");
}
```

타입 2. 리턴 타입이 void가 아닌 경우 + 입력 매개변수 X
```
int data() {
	return 3 //반드시 return값이 필요하다. 
}
```

타입 3. 리턴 타입이 void가 아닌 경우 + 입력 매개변수 O
```
double sum(int a, double b) {
	return a+b;
}
```

### 7.2.3 클래스 호출하기

- 상황 1. 클래스 외부에서 메서드 호출하기 → 객체 생성 먼저!
- 상황 2. 클래스 내부에서 메서드 호출하기 → 객체 생성 필요 없지롱 (단, static로 된 경우만)
- 상황 3. 입력매개 변수가 배열인 메서드 호출하기 → 입력 매개변수에 형식을 맞춘 뒤 값을 넘겨줘야한다. 이때, 값 객체가 전달되는게 아니라 위치값을 전달 하는 것이다.
	- 방법1. `참조변수 = new int[]{값, 값, 값}`로 참조변수를 전달
	- 방법2. 매개변수에 바로 `(new int[]{값, 값, 값})` 입력

기본자료형 입력매개변수 vs 참조자료형 입력매개변수
- 기본자료형을 입력매개변수로 전달하는 경우, 실제 값이 메서드의 지역변수로 복사되어 전달된다.
- 참조자료형을 입력매개변수로 전달하는 경우, 객체의 참조값을 복사하여 전달한다.

### 7.2.4 오버로딩된 메서드
메서드 시그니처 : 메서드명과 입력매개변수의 자료형
만약, 이 메서드 시그니처가 다르면, 메서드 이름이 같더라도 다른 메서드로 인식한다.
→ 메서드 오버로딩이 가능해진다!

### 7.2.5 가변 길이 배열 입력매개변수 메서드
입력매개변수가 너무 많은 경우, 개수에 상관없이 자료형에 따라 값을 받기위해 사용하는 방식
- 작성 방식 : `리턴타입 메서드명(자료형... 참조변수명) {}`

## 7.3 생성자
`객체를 생성하는 역할, 객체 내 필드의 초기화`

### 7.3.1. 생성자의 특징
```
클래스명(입력매개변수){

}
```
- 특징 1. 클래스명과 동일한 이름으로 지어야 한다.
- 특징 2. 리턴 타입이 없다.
- 입력 매개 변수는 생략 가능하다

### 7.3.2 기본 생성자의 자동 추가
생성자를 포함하지 않는 클래스에게도 컴파일러가 기본 생성자를 추가해준다.
- 기본 생성자 = 입력 매개변수가 없는 생성자

### 7.3.3 생성자의 객체의 생성 방법
생성자의 입력매개변수 자료형이나 개수에 따라 다양하게 객체를 생성할 수 있다. (메서드 오버로딩 개념과 유사함)

## 7.4 this 키워드와 this() 메서드
 원칙 `모든 사용할 수 있는 상태의 멤버는 항상 객체 속에서만 존재한다.`
 
### 7.4.1 내부 객체 참조 변수명인 this 키워드
this : 모든 메서드에 있는 자신이 포함된 클래스의 객체를 가리키는 참조 변수 
생략하면 컴파일러가 자동으로 작성해준다!
- 명시적으로 this를 작성해야 하는 경우 : 필드와 지역변수의 이름이 같은 경우에, 우선 지역변수로 컴파일러가 간주한다. 만약, 필드의 값을 쓰고 싶다면 this를 붙여주자

### 7.4.2 클래스 내 다른 생성자를 호출하는 this() 매서드
`자신이 속한 클래스 내부의 다른 생성자를 호출하는 명령`

- 규칙1. 생성자의 내부에서만 사용할 수 있다.
- 규칙2. 생성자의 첫 줄에 위치해야 한다.
