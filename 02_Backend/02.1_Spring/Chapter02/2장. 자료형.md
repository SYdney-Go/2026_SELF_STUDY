
## 2.1 변수와 자료형
### 2.1.1 자료형 선언하기
C나 자바 같은 컴파일 언어는 변수를 사용하기 전에 반드시 자료형을 선언해야한다. 이때, 자료형을 표기해야한다. 선언된 변수는 메모리에 변숫값을 저장할 수 있는 공간을 가진다.
- 자료형을 선언하지 않으면 사용할 수 없다.
- 하나의 소스코드 안에서 한번 선언된 자료형은 변경이 안된다.
### 2.1.2 변수 사용하기
```
변수 선언 → 값 대입
```
- 변수 선언 :  `자료형 변수명 = 값;`
- 값 대입 : `변수명 = 값;`
- 예시
``` 
// 방법1 //
int a;
a = 3;

// 방법2 //
int a = 3;
```

## 2.2 이름 짓기
### 2.2.1 이름을 지을 때 지켜야 하는 필수 사항
- 이름 짓기 대상 : 변수, 상수, 메서드
- 규칙1. 영문 대소 문자, 한글 사용 가능
- 규칙2. 특수문자는 _ 와 $ 만 허용 된다.
- 규칙3. 아라비아 숫자는 사욯 가능하나, 첫번째 글자로 올 수 없다.
- 규칙4. 자바에서 사용하는 예약어는 사용할 수 없다.

### 2.2.2 이름을 지을 때 지키면 좋은 권장 사항
변수명, 메서드 명
- 권장1. 영문 소문자로 시작한다
- 권장2. 영단어를 결합할 때는 새로운 단어의 첫 단어를 대문자로 한다.

상수명
- 권장1. 모두 대문자로 작성하되 결합할 때는 _ 를 사용한다.

### 2.2.3 변수의 생존 기간
```
선언 시점에 생성 → 자신이 선언된 중괄호가 닫히면 메모리에서 삭제
```

## 2.3 자료형의 종류
```
기본 자료형
├ boolean
├ byte, short, int, long
├ float, double
└ char
참조 자료형
└ 객체 : 배열, 클래스, 인터페이스
```

### 2.3.1 기본 자료형과 참조 자료형의 차이

|                 | 기본 자료형  | 참조 자료형                        |
| --------------- | ------- | ----------------------------- |
| 이름 규칙           | 소문자로 시작 | 대문자로 시작                       |
| 실제 데이터 값의 저장 위치 | 스택 메모리  | 힙 메모리 (스택 메모리에 힙 메모리 위치값을 저장) |

### 2.3.2 기본 자료형의 메모리 크기와 저장할 수 있는 값의 범위
＊ 패쓰. 그외 기타 자료형도? 패쓰. 다 똑같고 지겨움
## 2.4 기본 자료형 간의 타입 변환
boolean 빼고는 자료형의 타입을 변환할 수 있다.
```
자료형 A 변수명 = (자료형 A) 값
```

### 2.4.1 자동 타입 변환과 수동 타입 변환
- 자동 타입 변환 : 컴파일러가 자동으로 타입을 변환
- 수동 타입 변환 : 개발자가 직접 타입을 변환
- 업케스팅 : 크기가 작은 자료형 → 크기가 큰 자료형 대입 (문제 X)
- 다운케스팅 : 크기가 큰 자료형 → 크기가 작은 자료형 대입 (자동 타입 변환 X, 명시변환 필요!)

### 2.4.2 기본 자료형 간의 연산
같은 자료형 끼리는 연산이 가능하다!
다른 자료형 끼리도 연산이 가능하다. 작은 자료형을 자동 업캐스팅 하여 계산한다.
단, byte, short끼리의 연산이라도 결과는 int 자료형으로 나온다.
