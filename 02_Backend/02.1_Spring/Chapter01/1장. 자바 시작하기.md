## 1.1 프로그래밍 언어와 자바

### 1.1.1 프로그래밍 언어

`기계어 (0,1) ↔ 어셈블리어 ↔ 고급 언어` 

자바 같이 사람이 이해하기 쉬운 문장 언어를 컴파일러를 통해 컴퓨터가 이해할 수 있는 언어로 변환한다.

＊ 정확하게는 Java 소스 코드인 `.java` 파일을 `javac` 컴파일러가 JVM이 이해할 수 있는 `.class` 형태의 바이트코드로 변환한다. 이후 JVM(Java Virtual Machine)이 이 바이트코드를 읽고 실행하며, 실행 과정에서 운영체제와 CPU 환경에 맞는 기계어로 변환한다. 이 구조 덕분에 Java는 운영체제마다 서로 다른 실행 파일을 만들 필요 없이, 각 운영체제에 맞는 JVM 위에서 동일한 바이트코드를 실행할 수 있다.
### 1.1.2 자바의 역사
＊ 패스 하겠다. (관심없음)
### 1.1.3 자바의 플랫폼 독립성
자바의 가장 큰 특징이다. 

운영체제마다 실행 파일 형식과 실행 방식이 다르기 때문에, 네이티브 프로그램은 특정 플랫폼에 맞게 빌드되어야 한다. 예를 들어 윈도우는 `.exe`파일만, 맥은 `.app`, 리눅스는 `.sh`만 운영 프로그램 형식으로 지원하는 플랫폼 종속성을 가진다.

자바는 `.class` 라는 독립적인 실행 파일을 가지지만, 자바 가상 머신인 `JVM`이 각기 다른 운영 체제에서도 실행 될 수 있게 한다.

### 1.1.4 자바 개발 도구와 자바 실행 환경
자바 개발 도구 (JDK)
자바를 이용해 프로그램을 개발하는데 필요한 도구를 모아둔 집합
(개발자에게 필요한 패키지)

자바 개발 환경 (JRE)
컴파일된 자바 프로그램을 실행하기 위한 환경
(사용자가 프로그램을 실행하는 데 필요한 환경)

## 1.3 자바 프로젝트
### 1.3.1 자바 프로젝트 생성 및 실행하기
```
프로젝트 생성 → 패키지 생성 → 소스 파일 작성(.java) → 컴파일(.class) → 실행
```

패키지
관련있는 클래스를 묶어두는 폴더 개념으로, 하위 java 파일의 관리와 클래스 충돌을 방지하기 위해서 사용한다.

＊ 실습은 패스하겠다. 
## 1.4 자바 프로그램의 기본 구조
### 1.4.1 자바 소스 코드의 실행 과정
```
소스 파일 작성(.java) → 컴파일(.class) → JVM 메모리 할당 → main() 실행
```

컴파일이 실행되면 소스코드는 src 폴더에, 바이트 코드는 bin 폴더에 자동 저장된다. 바이트코드가 실행되면 JVM은 운영체제에서 메모리를 할당받아 프로그램을 실행한다.

이때, JVM은 메모리를 메서드 영역, 스택 영역, 힙 영역으로 나누어서 사용한다. 메서드 영역은 클래스, 정적, 상수 값을 저장하는 역할을 한다.

메모리 할당이 끝나면, 클래스 내 main() 메서드를 찾아서 실행한다.
### 1.4.2 소스 코드의 기본 구조 분석

```
package sec01_basicsyntax.EX01_JavaBasicStructure;

/*생성 프로젝트 기본 구조*/ 

/* ctrl+shift+'/'
     처음 만든 클래스
   (여러 줄 주석)
*/

public class JavaBasicStructure {	
	
	public static void main(String[] args) {
		
		//한줄주석 : 화면 출력 코드 ctrl+'/' 
		System.out.println("콘솔 화면에 출력");
	
	}
	
}
```

주석
주석 처리된 문장은 컴파일 할 때 제외된다.
- 1줄 쓸 때 `//주석내용//`
- 2줄 이상 쓸 때 `/＊주석내용＊/`

패키지 선언부
`package sec01_basicsyntax.EX01_JavaBasicStructure;` 
첫 줄에 반드시 패키지 선언이 있어야 함.
위 같은 경우, 해당 소스 파일이 `sec01_basicsyntax/EX01_JavaBasicStructure/` 안에 존재한다는 선언이다. 

클래스 
- public : 이 클래스는 다른 패키지에서도 사용 가능함을 명시. 단, 최대 하나의 클래스만 public으로 포함될 수 있음
- public이 붙은 클래스 명은 소스파일명과 일치해야 한다.
- 클래스 내부 구성요소 {}
	- 필드, 메서드, 생성자, 이너 클래스

main() 메서드
- 실행 이후 가장 먼저 실행되는 메서드
- 리턴 타입을 앞에 붙인다.

### 1.4.3 소스 파일 컴파일과 바이트 코드 생성
바이트 코드는 java 소스파일 내 클래스 갯수 단위로 생긴다.
- 퍼블릭 클래스
- 일반 클래스
- 이너 클래스 (아우터클래스를 꼭 같이 명시해야한다.)
