## 4.1 제어문
### 4.1.1 제어문 정의
프로그램의 실행 순서를 바꾸는 기능
### 4.1.2 if 선택 제어문
조건식에 따라 실행문의 실행 여부를 결정하는 선택 제어문

if 단일 구문
```
if(조건식 {
	실행 구문;	
}
)
```

if-else 구문
```
if(조건식) {
	실행 구문;
}
else {
	if 조건식이 false일 때 실행할 실행구문;
}
```

if-else if-else 구문
```
//조건식 중 하나라도 만족하면 제어문을 탈출한다//

if(조건식) {
	실행 구문;
}
else if(조건식2) {
	if 조건식이 false일 때 실행할 실행구문;
}
else {
	if 조건식과 else if 조건식2가 false일 때 실행할 실행구문;
}
```

### 4.1.3 switch 선택 제어문

```
switch(점프위치) {
case 위치1 : 
	실행 구문;
case 위치2 : 
	실행 구문;
case 위치3 : 
	실행 구문;
~
default:
	// 일치하는 위치값이 없을때 이동할 공간 //
	실행 구문;
}
```
이때, switch 문은 위치로 이동만 시켜주기 때문에 조건 만족 후 제어문을 벗어나고 싶으면 `break`를 써주어야 한다

### 4.1.4 for 반복 제어문

```
for(초기식; 조건식; 증감식) {
	// 조건식을 비워두면 컴파일러가 자동으로 true를 채운다 (무한동력) //
	실행구문;  // 조건식이 true 일때까지 실행.//
}
```

### 4.1.5 while 반복 제어문

```
초기식;
while(조건식) {
	실행 구문;
	증감식; // 이 증감에 따라 조건식이 true 인 동안 실행//
}
```

### 4.1.6 do-while 반복 제어문

```
초기식;
do {
	실행 구문; // 1회는 무조건 실행 //
	증감식;
} while(조건식);
```

## 4.2 제어 키워드
### 4.2.1 break 제어 키워드
가장 가까운 중괄호 탈출 키워드

중복 반복문 탈출 방법
```
out:
for(초기식; 조건식; 증감식) {
	for(초기식; 조건식; 증감식){
		실행 구문1;
		break out; //실행 구문2에 묶여있는 조건식과 관련없이 out 까지 탈출한다.//
	}
	실행구문2;
}
```

### 4.2.2 continue 제어 키워드
반복문 실행 중, continue를 만나면 남은 실행 구문을 무시하고, 다음 반복을 위한 증감식 수행 
단독으로 쓰이기보다는 특정 조건일때, continue를 실행하는 방식으로 많이 쓰임
